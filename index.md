# Data Description
The following datasets will be used to model correlations between golden eagle (*Aquila chrysaetos*) population trends and occupancy status with landscape scale environmental factors including primary production, drought, fire, and human development in Eastern Oregon. 

The first dataset includes is a spreadsheet of observations from known golden eagle nesting areas and will include all historic surveys (2011-2020) as well as current surveys (2024 - present). The spreadsheet includes nest IDs, nest locations (GPS coordinates), occupancy status (eagle presence), and reproductive outcome (success, # chicks fledged). The historic data was collected by Oregon Department of Fish & Wildlife (ODFW) personnel on paper data sheets and compiled into a CSV spreadsheet. Current data will be collected by field technicians visiting nest sites multiple times per breeding season and entering observations in ESRI Field Maps form. The observations in Field Maps are synced to Arc GIS Online server and will be downloaded and saved as CSV files, appended to historic data, and compiled by year. Golden Eagle Nest Data – 1-2 GB

The second dataset is a collection of GPS data points from 25 individually marked eagles in the study area. This data is collected via MTI GPS/UHF solar-charged backpack transmitters that are placed on captured eagles. The transmitters upload the data to ARGOS satellites, from which location data can be downloaded as a CSV for each transmitter. Data will be processed and combined in R software to model home range parameters of breeding eagles in Oregon. Golden Eagle GPS Data – 2-5 GB

The third dataset is a spreadsheet of remote sensed environmental data collected from publicly available resources including MODIS, TerraClimate, and Landsat. The raster files are pulled from publicly available resources, clipped to the home range of a nesting golden eagle (modeled above) around each nest site, and averaged over the appropriate area and timeframe. This data will be linked to each individual nest ID from the previous dataset. This data will be sourced and processed by utilizing Google Earth Engine before downloading CSV files and compiling data by nest ID and year. Remote Sensed Environmental Data – 1-2 GB


# Roles and Responsibilities
The golden eagle nest, and GPS data is collected and compiled by ODFW and used by the principle investigator and student for analysis. It is the role and responsibility of the department to generate, document, protect, archive, preserve and share the data. This includes any instrumentation and software maintenance needed to collect or preserve data.

Remote sensed environmental data is managed and compiled by MODIS, TerraClimate, and Landsat database managers. They are responsible for the generation, documentation, protection, archiving, and preservation of data. Data will be sourced via Google Earth Engine, which is responsible for the software necessary to acccess and retrieve the data.

Within the scope of the project anaylsis, it is the responsibility of the PI and the graduate student to implement the data management plan and to coordinate roles and responsibilities of outside collaborators and staff.

The graduate student will gather data generated from publicly available remote sensed databases and compile with ODFW datasets. It is the graduate student’s responsibility to ensure all current and past data is organized and checked for errors. The student will compile and generate metadata for each data source. The graduate student is responsible for organizing, naming, preserving, sharing, and version control of all files, scripts, code and any products of data analysis. All analysis will be the responsibility of the PI and graduate student.

Original nest, GPS, and remote sensed data will be stored, protected, and backed-up on the respective organizations' database. All field (historic and current) and remote sensed data will be accessed via network shared drives, compiled, and stored on the graduate student’s local hard drive and automatically backed up on lab network shared drive. All analysis, code and data files will be processed and version controlled via Github repository.


# Data Standards and Metadata
Data will be formatted as CSV to ensure interoperability, and all scripts and code will be in R files. File structure will reflect the purposes of each data source; raw data, code/analysis, presentations/output with corresponding README files in each folder to describe what files are contained in each.

All metadata associated with tabular data will be generated and documented in separate documents within the Github repository folder associated with the original data source (nest data, GPS data, habitat data). There will be both human (html) and machine readable (XML) files for the metadata files. Metadata generation will follow Ecological Metadata Language (EML). While not exhaustive, each data source’s metadata will include: author(s), licensure needed, brief description, original data source, date accessed/created, methodologies for data collection/manipulation, temporal and spatial scale of data, data attribute descriptions, and units. Metadata will be created while data is sourced and collected and updated as necessary through the project when new data is added.

All metadata associated with code will be generated and documented in a separate document within the Github folder associated with all code and analysis done for the research. Metadata will include author(s), purpose, packages used, description of steps for analysis, and anticipated output including type and location of output file. Metadata will be created during coding and analysis to best encompass processes. 


# Storage and Security
Data used during analysis will be stored on private Github repository as well as local drive with automatic backups onto network shared drive. All data, code and metadata generated for my project will utilize Github repository for version control. 

Data will follow a strict naming convention for all files to maintain organization. Files will be named succinctly with what is contained within each CSV or R file (i.e. goea_nestdata, goea_gpsdata, goea_habitatdata, etc.). Any manual version control from downloading data from outside sources will be clearly documented with a date at the end of each file (i.e. goea_gpsdata_20260430, goea_gpsdata_20260501), in addition to a subscript if there are multiple files on the same day (i.e. goea_gpsdata_20260430_a, goea_gpsdata_20260430_b).


# Access and Data Sharing
All nest analysis data from my project will be publicly available without restrictions or permissions to access, as it is collected under a U.S. Fish & Wildlife Service grant which must follow federal government regulations. Location data supporting my research are sensitive and not available publicly as directed by state law (ORS 496.182). The research contains data of a protected species listed that has a history of harm and known demand for illegal harassment and contains sites used for sensitive or vulnerable life stages. The location data may be shared with the federal government, tribal governments, a public utility or an accredited university by contacting Oregon Department of Fish and Wildlife. Location data will be aggregated to protect specific locations that may cause harm to breeding or nesting eagles and included in the publicly available dataset. Portions of my data set are taken from other publicly available data sources and altered for my data analysis, therefore they are not protected or restricted.

There will be no license attached to any of the data sets created in my project and it will not contain any restrictions for the reuse, redistribution or creation of derivatives. Data will be made available at publication of the research. All publicly available data and metadata from the project will be accessed via Dryad repository. A link to the repository with a digital object identifier will be published in all reports, papers and articles derived from the analysis. Data will be made available in downloadable CSV files, as well as shapefile and KML files for all aggregated location-based data. All code used for analysis will be made available as R and txt files.

# Archiving and Preservation
All publicly available data will be preserved and archived in the Dryad repository indefinitely. This includes all CSV files, shapefiles, and code included to analyze data. There will be open access to archived data in the Dryad repository via the DOI assigned at publication. All location data will be preserved indefinitely in internal ODFW database and accessed via permissions from department database manager. Data collection efforts are also done in collaboration with the U.S. Fish and Wildlife Service and will be archived on the internal ServCat database system, with access via data.gov.
